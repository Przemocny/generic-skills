# Report Templates

Szablony raportów dla różnych scope researchu.

---

## Minimal Template

Dla prostych tematów (quick overview, single focused question).

```markdown

**Data:** [YYYY-MM-DD]
**Autor:** Claude Code (General Research Skill)

## Executive Summary
[2-3 zdania: o co pytał użytkownik i główne wnioski]

## Key Findings

[Najważniejsze informacje które znalazłeś, pogrupowane logicznie]

## Recommendations

**Dla [kontekst użytkownika]:**
1. [Konkretna rekomendacja]
2. [Konkretna rekomendacja]

## Sources

1. [Tytuł](URL) - [typ źródła]
2. [Tytuł](URL) - [typ źródła]
```

## Template selection guide

**Use Minimal when:**
- Simple, focused question
- Quick overview needed
- Limited scope (1-2 aspects)
- 3-5 sources sufficient

**Use Standard when:**
- Balanced research (most cases)
- Multiple aspects to cover
- Best practices needed
- 5-8 sources
- Decision-making context

**Use Extended when:**
- Complex, multi-faceted topic
- Critical business/technical decisions
- Need comprehensive analysis
- Multiple alternatives to compare
- 8+ sources
- Long-term implications

## Report writing tips

**Executive Summary:**
- Answer: What did user ask? What did you find? What should they do?
- Keep it actionable
- 2-3 sentences max

**Findings:**
- Group by logical themes/aspects
- Lead with conclusions, then details
- Always cite sources

**Patterns & Consensus:**
- This is your analysis, not just data collection
- Point out what's agreed vs. debated
- Explain WHY disagreements exist (context)

**Best Practices:**
- Explain the "why", not just "what"
- Include context: when to use, when not to use
- Cite authoritative sources

**Recommendations:**
- Make them actionable
- Prioritize (High/Medium/Low)
- Include concrete next steps
- Acknowledge trade-offs

**Sources:**
- Always include tier level
- Note the type (docs/blog/case study)
- Order by importance/tier
